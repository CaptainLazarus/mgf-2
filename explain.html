Linear Scan — How close works

Every function in linear.ml (seed, left_boundary_fill, right_boundary_fill, combine) uses the same internal pattern: a worklist + seen loop called close. Here's what it does and why it's written the way it is.

The Problem: same item, multiple trees

A state is a list of (h_item, forest_node list). The same h_item can be produced by multiple derivation paths — that's ambiguity. We need to collect all trees for the same item into one entry.

Example: two paths both produce I(1,1,2)
I(1,1,2) — FNode[FLeaf "w1"; FVirtual X]
I(1,1,2) — FNode[FLeaf "w1"; FVirtual Y]

→ should become one entry: I(1,1,2) — [tree1, tree2]

The old code used List.mem_assoc which just skipped the second entry entirely — dropping tree2. That's a bug.

The Fix: merge_into

let merge_into seen item nodes =
  match List.assoc_opt item seen with
  (* item not seen before → just add it *)
  | None → ((item, nodes) :: seen, nodes)
  (* item already seen → merge only the new nodes *)
  | Some existing →
    let new_only = List.filter (fun n → not (List.mem n existing)) nodes in
    let seen' = List.map (fun (i, ns) → if i = item then (i, ns @ new_only) else (i, ns)) seen in
    (seen', new_only)

It returns two things: the updated seen, and only the genuinely new nodes. Why only new nodes? Because we already fired projections for the existing ones — no need to do it again.

The close loop

let rec close worklist seen =
  match worklist with
  | [] → seen
  | (item, nodes) :: rest →
    let (seen', new_nodes) = merge_into seen item nodes in
    if new_nodes = [] then close rest seen'  (* nothing new, skip *)
    else
      let projected = ... fire projections on new_nodes only ... in
      close (projected @ rest) seen'
Casenew_nodesWhat happens
Item is brand new= nodes (all of them)Add to seen, fire projections on all nodes
Item seen, genuinely new trees= subset of nodesMerge into seen, fire projections on new trees only
Item seen, no new trees= []Skip entirely

Summary

  • worklist = things to process
  • seen = the growing state (the result)
  • merge_into = add item to seen, return only new trees (to avoid re-projecting)
  • Projections fire only on new trees, not on trees already in seen
  • Loop ends when worklist is empty → seen is returned
